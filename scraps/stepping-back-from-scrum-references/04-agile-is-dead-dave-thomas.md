## 書き起こし

13年前、私は17人の中年白人男性の一人として、ユタ州スノーバードに集まりました。私たちはソフトウェア開発に関する共通の信念を持っており、その信念を言葉で表現できないかと考えていました。

価値観のリストをまとめるのに1日もかかりませんでした。私たちはその価値観と実践のリストを、「アジャイルソフトウェア開発宣言」として発表しました：

| | | |
|-----------:|:-|:---------|
|**個人と対話** |> |プロセスとツール |
|**動くソフトウェア**|> |包括的なドキュメント|
|**顧客との協調** |> |契約交渉 |
|**変化への対応** |> |計画に従うこと |

私はこのプロセスとその成果の両方を誇りに思っています。そしてこのマニフェストの存在が、開発者たちを80年代・90年代の無駄で精神を破壊するような慣行から解放する助けになったと思っています。

しかし、スノーバードの会合以来、私はいかなるアジャイルのイベントにも参加せず、アジャイルアライアンスに加入せず、「アジャイル」コンサルティングも行っていません。10周年記念のお祝いにも参加しませんでした。

なぜか？それらはいずれも、私たちが作ったマニフェストの精神に沿うものではないと感じたからです。アジリティについてカンファレンスを開催することは、バレエのカンファレンスを開くことと大して変わりません。4つの価値観の周りに業界団体を作るのは、「呼吸する人々の組合」を作るようなものだと常々思っていました。

そして残念ながら、時間が私の正しさを証明してしまいました。「アジャイル」という言葉は、実質的に無意味になるほどまで歪められ、アジャイルコミュニティと呼ばれるものは、コンサルタントやベンダーがサービスや製品を売りつける場に成り果てています。

そろそろ「アジャイル」という言葉を引退させる時が来たと思います。

名詞として使われる場面での禁止なら、誰も異論はないでしょう。それは単純に間違っています。「アジャイルを正しくやれ」とか「バカでもわかるアジャイル」といった表現は、英語への無数の攻撃のほんの一例です。これらは無意味です。アジャイルは名詞ではなく形容詞であり、必ず何かを修飾しなければなりません。「アジャイルを正しくやれ」は「オレンジを正しくやれ」と言うようなものです。

しかし文法の問題を超えて、もっと大きな問題があります。マニフェストが広まるにつれ、「アジャイル」という言葉は、主張を述べたい人、請求書を切りたい人、製品を売りたい人を引き寄せる磁石になってしまいました。「エコ」「ナチュラル」といった言葉が商業利用されるように、販売促進のためのマーケティング用語になってしまったのです。このように乱用された言葉は無用になります――ブランドへと変質する過程で、意味を失っていきます。

これは皆に害をもたらしますが、私は特に開発者へのダメージを気にかけています。コードを書くことは簡単ではなく、開発者はより効果的に価値を届けるためのものを自然と探し求めています。マニフェストの価値観と実践に従うことが、その取り組みを助けると私は今でも固く信じています。

しかし「アジャイル」という言葉が無意味になってしまうと、開発者はもはや自分の実践において何が有用かを見極める指針として使えなくなります。「アジャイル」という単語を空白に置き換えても、何も変わらないのと同じです。

### 右側への移動

4つの価値観をもう一度見てみましょう：

個人と対話 ＞ プロセスとツール
動くソフトウェア ＞ 包括的なドキュメント
顧客との協調 ＞ 契約交渉
変化への対応 ＞ 計画に従うこと

左側の言葉は理想を表しています――左右どちらかを選ぶ場面で、アジリティを持って開発する者は左を選びます。

では、「アジャイルを導入します」と言うコンサルタントやベンダーを見てください。彼らが左右の軸のどこに位置しているかを自問してみてください。おそらく、プロセスとツールを重視し、多くの成果物（マネージャーを喜ばせるための文書を指すコンサルタント用語）を提案し、ホワイトボードと付箋紙よりはるかに多くの計画を立てていることに気づくでしょう。

もしそう感じるなら、それは「アジャイル」という言葉の腐敗と価値低下のさらなる証拠です。

（もちろん、これらのコンサルタントの中には2日間のトレーニングコースを受けた人もいるでしょう。私は受けていないので、彼らはマスターで私はそうではありません。だから私が間違っているのかもしれません。）

### 基本に立ち返る

アジャイルな方法で何かをするとはこういうことです：

何をするか：
- 今いる場所を把握する
- ゴールに向けて小さな一歩を踏み出す
- 学んだことをもとに理解を調整する
- 繰り返す

どうやるか：
ほぼ同じ価値をもたらす2つ以上の選択肢に直面したとき、将来の変更をより簡単にする道を選べ。

以上です。この4行と1つの実践が、効果的なソフトウェア開発に関する知識のすべてを包含しています。もちろん、これにはかなりの思考が伴い、変数の命名から長期的なデリバリーまで、あらゆるレベルでこの基本ループが入れ子状に繰り返されます。しかし、それより大きく複雑なものを提示する人は、あなたに何かを売りつけようとしているだけです。

これらの文はすべて命令形です――何をすべきか、どうすべきかを動詞で示しています。

だからこそ、私の提案はこうです。

「アジャイル」という言葉は、何もしない人たちに譲りましょう。

代わりに、私たちがすることを表す言葉を使いましょう。

### アジリティをもって開発しよう

- あなたはアジャイルプログラマーではなく、アジリティをもってプログラミングするプログラマーです。
- あなたはアジャイルチームで働くのではなく、アジリティを発揮するチームで働きます。
- アジャイルツールを使うのではなく、アジリティを高めるツールを使います。

「アジャイル」という言葉を何にでも貼り付けるのは簡単です。「アジリティ」はそう簡単には悪用できません。

それは重要なことです――ラベルは売り買いできます。短期コースを受ければ、職肩書きにラベルを追加できます。しかし経験は買えません――稼ぐしかないのです。

### 私たちの投資を守ろう

結局、私たちが何をするかが、何と呼ぶかより大切です。しかし良い言葉は効果的なコミュニケーションを助けます。

私たちは「アジャイル」という言葉を失いました。「アジリティ」は守り続けましょう。意味を保ち続け、私たちのアイデアの魂を奪って売り返そうとする人たちから守りましょう。
